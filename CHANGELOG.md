# 變更紀錄

本檔案記錄原型每個版本的變動。

## [v0.12] — 2026-06-25

### 修正（預設月份改以「當前日曆月」為準）
- **問題**：開啟專案時預設跳到資料的「最後一個月」，但月報資料含未來月份（排到專案完工），導致一開就停在未來月份。
- **修正**：比照 Power Apps，預設停在「當前日曆月份」`EOMonth(today,0)`。
  - 編輯權限改綁當前月：僅當前月可填寫；歷史月份顯示「歷史月份唯讀」、未來月份顯示「未來月份唯讀」橫幅。
  - 開啟專案的預設選月：優先當前月；無當前月資料則退而選「不晚於當前月」的最新一筆；全為未來則取最早。
  - 近三月檢視改以當前月為基準（前兩月＋當月），缺資料的月份顯示「—」。
- **示範資料產生器**：改為動態生成「最近 12 個月（結束於當前月）」，讓公開 demo 永遠停在可編輯的當月。

## [v0.11] — 2026-06-24

### 新增（完整對齊 Power Apps 雙畫面）
- **近三月唯讀檢視**：頂部新增「填寫當月 / 近三月檢視」兩個檢視頁籤，比照 PA 的 EditScreen + ReadOnlyScreen。
- 近三月檢視為唯讀對照表：列出最近三個月（舊→新）的月報數字（累計收入/投入、本月預定投入/實際投入預估、年度累計預定/實際投入），當月欄位以淺藍高亮；下方附當月兩段敘述（唯讀）。
- 檢視模式下自動隱藏填寫流程的鎖定切換鈕與橫幅；切換專案/月份時檢視即時更新。
- 註：PA ReadOnlyScreen 實際版面尚未取得截圖，此為合理推斷的對照式呈現，待後續以 PA 畫面微調。

## [v0.10] — 2026-06-24

### 新增（XSS 防護）
- **富文本白名單清洗 `sanitizeHtml()`**：兩段 HTML 欄位（投入與收入說明、警示及具體做法）存檔前先做前端白名單清洗——移除 `<script>/<iframe>/<style>` 等危險標籤、所有 `on*` 事件屬性、`javascript:`／`data:`（非圖片）等不安全 URL、不在白名單的 style 屬性；保留安全格式（粗體/斜體/清單/連結/安全顏色）。
- 清洗到不安全內容時跳出提示「已移除不安全的 HTML 內容」。
- 純原生 JS（無 DOMPurify 依賴，配合資安不可裝套件）；定位為**前端第一道防線**，正式版後端仍會以 HtmlSanitizer 再清一次（雙層）。
- 測試套件 G5 的 4 條 XSS 案例由「政策待實作」轉為**已實作並瀏覽器實測通過**（29/29 全綠）。

## [v0.9] — 2026-06-24

### 新增（互動狀態，讓原型從「會顯示」變「會操作」）
- **月結鎖定 / 唯讀切換**：可模擬月結鎖定當月，鎖定後整月轉唯讀（紅色橫幅、所有可編輯欄位 + 存檔鈕禁用、富文本工具列灰化）。鎖定狀態以 `(專案,月份)` 為 key 持久化。
- **歷史月份唯讀**：比照 Power Apps，僅「最新月份」可填寫；切到歷史月份顯示黃色「唯讀檢視」橫幅並禁用編輯。
- **本月實際收入欄位驗證**：非負整數檢核（負數→「不可為負數」、小數→「請輸入整數」），無效時即時顯示紅色錯誤標籤、輸入框紅框、並禁用存檔鈕。空值允許（非必填）。
- **專案下拉過濾**：排除已結算/已結案專案，只列進行中（對齊「10–20 個進行中」需求）。

### 修正
- 修正輸入體驗 bug：先前每次按鍵都用 `render()` 重設輸入框值，導致小數中間值被吃掉、富文本游標跳動。改為「正在編輯（focus）的欄位不覆寫」，保留使用者輸入並讓衍生計算即時更新。

## [v0.8] — 2026-06-24

### 變更（複刻 Power Apps 版面）
- 全面重構版面以 1:1 對齊既有 Power Apps 介面佈局。
- **財務網格卡片化**：白底圓角卡片,灰底頁面背景,與 PA 一致。
- **日期列改為 4 欄**：預定開工日、預定完工日、展延後完工日、累計投入%——第 4 格置右上角與 PA 相同。
- **本月實際收入 inline 化**：從獨立區段搬進財務網格最底列,PA 風格的標籤 + 輸入框 + 存檔按鈕。
- **MonthlyEstimatedCost 收進摺疊區**：正式版自動讀取,開發模式才展開手動輸入。
- **付款表格簡化為 5 欄**：合約名稱、廠商名稱、計價金額、計價期別、計價日期——與 PA 完全對齊。
- **付款統計改為卡片內 4-grid**：與財務網格視覺語言一致。
- **富文本區段卡片化**：投入與收入說明、警示及具體做法改為白底卡片 + 工具列。
- **整體色彩更新**：背景 `#f3f2f1`(Fluent UI neutral)、文字 `#323130`、主色 `#0078d4`,統一 Microsoft 設計語言。

  - monthly estimated cost 待開放底層快照 view 的 SELECT 權限後自動讀取。

- 後端寫入表 `MonthlyReportDesc`(別名)確認最終結構:12 欄、主鍵 CLUSTERED on (ProjectID, MonthEnd)、所有欄位 default 設好(`TargetAmount=0`、`Active=1`、`CreateDate=getdate()`)。

### 文件
- 補上 `MERGE` UPSERT 範本至 `docs/BUSINESS-LOGIC.md`(供後端團隊參考)。

## [v0.4] — 2026-06-22

### 變更(對齊既有系統公式)
- 新增第 4 個可編輯欄位:**Monthly estimated cost**(`MonthlyEstimatedCost`),含獨立存檔按鈕與「未儲存」標籤。
- 修正多項計算公式以對齊既有系統:
  - **累計實際投入** = 本月 `AccRealCost` + 本月 `MonthlyEstimatedCost`(使用者輸入)
  - **年度累計實際投入** = `YearRealCost` + 本月 `MonthlyEstimatedCost`
  - **年度投入差** = (顯示年度累計實際投入) − `YearEstCost`(超支為正)
  - **累計投入%** = (顯示累計實際投入) / `BudgetRevisedAmt`
  - **本月實際投入(預估)** 由 `RealCost` 改為使用者輸入的 `MonthlyEstimatedCost`
  - **本月達成率** = `MonthlyEstimatedCost` / 本月 `EstCost`(原本誤用 `YearCostExecRate`)
- 「累計實際收入」公式維持 `上月 AccRealAmt + 本月 TargetAmount`(經驗證正確)。
- 開發資訊面板擴增,同時顯示收入側與投入側兩條計算過程,並提示 schema 待加欄位。

### 已知缺口
- `MonthlyReportDesc` schema 目前沒有 `MonthlyEstimatedCost` 欄位,需後端團隊新增。前端寫回模擬會印 payload,等 schema 補齊即可真實寫入。
